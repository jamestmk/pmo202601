# PMO 平台 V2 需求调整方案

> 基于 MVP 试运行反馈，对角色、权限、功能模块的调整。
> 确认后作为代码改造的依据。

---

## 1. 角色体系调整

沿用 ADMIN 作为最高权限角色（老板使用此角色），其余角色调整如下：

| 全局角色 | 说明 | 变化 |
|---------|------|------|
| ADMIN | 管理员/老板，全局可见，看板 + 薪资全览 | 不变，老板用此角色 |
| PROJECT_MANAGER | 项目经理，管项目进度 + 查看薪资 | 不变 |
| PRODUCT_MANAGER | 产品经理，提需求为主 | 不变 |
| DEVELOPER | 研发（多技术栈） | 不变 |
| TESTER | 测试 | 不变 |
| ANNOTATOR_LEAD | 数据标注组长 | **新增** |
| ANNOTATOR | 数据标注员 | **新增** |

项目级角色保持 MEMBER / PM 不变。

---

## 2. 权限调整

### 2.1 状态流转 — 全员可操作
- **任何项目成员**都可以变更需求的研发状态（不再限制为当前承接人 + PM）
- 流转时仍需指定下一承接人

### 2.2 项目创建 — 全员可操作
- **任何登录用户**都可以新建项目
- 任何登录用户都可以配置项目的研发状态流程
- 项目成员管理仍由项目创建者或 ADMIN 操作

### 2.3 数据可见性
- 每个角色只能看到自己有权限的数据
- ADMIN 可看所有项目、所有薪资
- PROJECT_MANAGER / ANNOTATOR_LEAD 可看所管项目的薪资
- 其他角色只能看自己的薪资
- 对话机器人遵循当前用户的数据权限

---

## 3. 新增功能模块

### 3.1 薪资管理

**录入方式**：项目经理 / 标注组长手动录入每人每笔金额

**数据字段**：
- 关联项目
- 关联成员
- 金额（人民币）
- 发放周期/日期
- 备注
- 状态：待审批 → 已确认 → 已发放

**薪资看板（北极星指标）**：
- 各项目薪资支出总额
- 人均产出（完成需求数 / 人数）
- 单需求成本（项目薪资总额 / 完成需求数）
- 人效趋势（按月/周）
- 后续目标：基于历史数据，为类似项目提供参考报价

**可见性**：
- ADMIN：所有项目、所有人的薪资数据
- PROJECT_MANAGER：自己管理的项目的薪资数据
- ANNOTATOR_LEAD：自己管理的项目/组的薪资数据
- 其他角色：仅自己的薪资明细

### 3.2 AI 需求归类

**触发方式**：
- 手动：提交需求时点击"AI 归类"按钮
- 自动：需求创建后自动调用（可在项目设置中开关）

**归类维度（全局统一）**：
- 需求类型（功能需求 / Bug修复 / 优化改进 / 数据标注 / 技术债务 / 其他）
- 模块/领域标签（可扩展）
- 建议优先级
- 预估复杂度（简单 / 中等 / 复杂）

**实现方式**：
- 后端调用大模型 API（具体模型待定）
- 基于需求标题 + 描述进行分析
- 结果写入需求字段，人工可覆盖

### 3.3 全局对话机器人

**入口**：页面右下角常驻悬浮按钮，点击展开对话面板

**能力范围**：
- 查询：项目状态、需求统计、个人任务、薪资信息
- 操作：创建需求、变更状态（确认后执行）
- 遵循当前登录用户的数据权限

**示例对话**：
- "XX 项目还有多少 P0 没处理？"
- "帮我在 XX 项目需求池加一条：用户登录页需要支持手机号"
- "我这个月的薪资发了多少？"
- "所有项目的交付率排名"（仅 ADMIN 可查全局）

**实现方式**：
- 前端对话 UI 组件
- 后端 API 接收自然语言，调用大模型解析意图
- 将意图映射到现有 Server Actions 执行
- 返回结构化结果 + 自然语言回复

## 4. 现有功能调整

### 4.1 需求字段扩展
新增以下字段（对应 AI 归类 + 薪资人效计算）：
- `category`：需求类型（全局统一枚举）
- `module`：模块/领域标签
- `complexity`：预估复杂度
- `aiClassified`：是否经过 AI 归类（布尔）

### 4.2 总看板增强
- 增加薪资相关的北极星指标卡片（按权限显示）
- 增加人效指标：人均产出、单需求成本

### 4.3 导航调整
- 新增"薪资"导航入口（按角色权限显示）
- 新增对话机器人悬浮入口（全局）

---

## 5. 实施优先级建议

| 优先级 | 模块 | 说明 |
|--------|------|------|
| P0 | 角色体系 + 权限调整 | 基础设施，其他功能依赖 |
| P0 | 状态流转/项目创建权限放开 | 改动小，立即生效 |
| P1 | 薪资管理 + 薪资看板 | 核心新功能 |
| P1 | 需求字段扩展 + AI 归类 | 提升效率 |
| P2 | 全局对话机器人 | 锦上添花，依赖前面的数据基础 |
